# 사업 배경 요약 (새 세션에 붙여넣기용)

## 누구인가
공주대 부동산학과 1학년. 시행사(개발), 투자 리서치, 기관급 시장분석 쪽 커리어를 목표로 함.
장기적으로 대한민국 대표 부동산 인프라 기업("부동산 산업의 운영체제")을 만드는 것이 목표.

## 최종 확정된 사업 방향
**AI 부동산 인텔리전스 플랫폼 — 부동산 PF(프로젝트 파이낸싱) 사업성 심사 자동화 SaaS**

- 처음 구상은 "개인 재건축·재개발 투자자용 AI 리포트"였으나, 경쟁사 분석 후 피벗함
- 경쟁사(랜드북/스페이스워크, 디스코, 밸류맵, 부동산플래닛)는 전부 B2C·공인중개사 대상
  주소검색 → 실거래가/사업성 리포트 서비스를 이미 제공 중 — 이 카테고리는 레드오션
- 디스코(10년 업력, 누적투자 73억, 직원 12명, 최근 성장률 0%)가 보여주듯 이 시장은
  "100조 규모 독점 인프라"가 아니라 스몰~미드 마켓 비즈니스로 이미 검증 끝난 상태
- **피벗 결론**: 첫 타겟 고객을 개인 투자자가 아니라 **신탁사·캐피탈사·저축은행·
  자산운용사의 PF 심사역**으로 변경. 이들은 심사 리스크가 크고(건당 수십~수백억),
  소프트웨어 예산도 크며, 기존 프롭테크가 건드리지 않은 공백 시장.

## MVP 정의
주소 검색 → 데이터 자동 수집 → AI 분석 → PF 심사용 투자 리포트 생성 → PDF 다운로드.
그 외 기능(앱, 커뮤니티, 자체 거래, 대출, SNS, 광고)은 초기에 전부 제외.

  대상이라 저장소엔 없음) 자동 조회, 실패 시 가정치로 폴백 + 어느 쪽을 썼는지 항상 배지로 표시

  고성군) 시/도 구분 처리 포함. 주소 입력 즉시 인식된 시군구를 화면에 표시(네트워크
  호출 없이, 오타를 분석 실행 전에 미리 잡아줌)
- **채점 모델**(`src/lib/scoring/index.js`, 현재 v1.3.0): 4개 카테고리 가중합산 100점
  (금융안정성40·사업성30·사업안정성20·입지경쟁력10) → AAA~D 20단계 등급
  - **하드 게이트**: 금융 4항목 중 2개 이상 위험, 또는 인허가·시행사·시공사 전부
    위험이면 총점과 무관하게 등급을 BB(투기적) 이하로 강제 캡. 가중합산만으로는
    한쪽 카테고리가 완전히 무너져도 나머지가 가려줘서 등급이 부풀려지던 결함을
    실제 검증(테스트 케이스 실행)으로 발견해 수정함
  - 신용보강구조(책임준공확약/조건부채무인수 등) 채점 항목 포함, 시공사 등급과는 별개 축
  - 이중계상 정리 완료(예상 분양성·주변거래·교통접근성·생활인프라 등 삭제/통합)
  - 각 항목마다 근거 문구(reason) 제공, 배점 손실 큰 순 리스크 TOP·강점 TOP 자동 추출
    (무작위 리스크 매트릭스는 삭제하고 실제 채점 데이터 기반으로 교체함)
- **리포트 구성**: 표지 → Executive Summary → 1.사업개요 → 2.시장성분석(입지·공급경쟁·
  실거래comps를 한 섹션으로 통합 + 분양경쟁력/공급리스크 자동 라벨·시장의견 문장 생성)
  → 3.금융개요(+PF자금용도) → 4.사업성지표 → 5.종합평가항목 → 6.리스크분석(위험/보통
  판정 전체를 배점손실 순으로 나열) → 분양률/금리/공사비 3종 스트레스테스트 →
  7.AI종합의견. 인쇄(PDF) 시 표지가 자동으로 별도 페이지로 분리되고, "요약만 보기"
  상태에서 눌러도 항상 전체 리포트로 출력됨
- **총사업비 입력**: 토지매입비 총액 직접입력 가능(비우면 자동계산), 소프트코스트를
  항목별 7종(철거비·설계비·감리비·분양마케팅비·각종부담금·예비비·기타) 또는 비율(%)
  입력 중 선택. 기존 "총사업비 직접입력"(최우선 적용)과 호환
- **분석 이력**: 실행할 때마다 localStorage에 자동 저장, 목록 조회·주소 검색·개별
  삭제·불러오기(폼만 복원)·재분석(폼 복원 즉시 재실행) UI 제공
- **PF 사례 검증 구조**: `data/pf-cases/`에 스키마(README.md) + 더미 사례 5건(우량성공/
  보통성공/지연/부도/정보부족 시나리오). 향후 AI 분석결과 vs 실제결과 비교 기능의
  자리만 만들어둔 상태(비교 로직 자체는 아직 없음)

## 진행 상황 (2026-07-22 대규모 세션)
이전 세션에서 미완성으로 남아있던 scoring 모듈 연동(계산 로직은 다 있는데
PFReportMVP.jsx가 삭제된 함수를 호출해 빌드가 깨진 상태)을 마무리한 뒤, 사용자가
지시한 "추가 개발 작업" 10개 항목 + 채점모델 검증 리포트 1개를 순서대로 처리했다.

   갱신돼 있어서 "서울만 지원"으로 잘못 알고 있었던 것을 재확인 후 정정)
2. 시장성 분석 섹션 추가
3. 총사업비 입력 고도화(항목별 세분화)
4. 분석 이력 기능 완성 (목록/검색/삭제/불러오기/재분석)
5. PDF 보고서 고도화(표지 신설·섹션 순서 재배열·리스크분석 섹션 신설·출력 품질)
6. `data/pf-cases` 구조 신설(더미 5건)
7. 주소 기반 시군구 자동 인식 표시

9. 코드 품질 개선(중복 제거, 컴포넌트 분리, JSDoc 타입 힌트)

    항목이 "데이터 가용성"만으로 등급을 좌우하는 과대평가를 발견 → 배점 조정
    (comps 9→6점, supplyCompetition 5→8점, v1.3.0으로 반영)

전부 기능별 커밋으로 나눠 `main`에 푸시 완료. 매 커밋마다 `npm run build` +

scoring 엔진 로직 변경 없음.

탭 이동 시 자동 조회되는지).

## 자기자본비율 기본값 처리 수정 (v1.4.0 → v1.4.1, 2026-07-22)
위에서 남겨둔 "재무조건 미정 시 안전해 보이는 기본값 때문에 리스크를 놓치는" 구조적 위험 후속
조치. 자기자본비율 기본값(20%)이 "우수" 컷오프(20% 이상)와 정확히 같아서 미입력 상태로 두면
항상 만점을 받던 걸 발견 — 시행사·시공사 등 정성 항목은 "미정(정보 없음)"이면 위험 처리되는데
이 정량 항목만 그런 로직이 없었음. `equityRatioIsDefault` 플래그 추가해 미입력 시 위험 처리로
통일(`src/lib/analysis.js`·`src/lib/scoring/index.js`, `SCORING_MODEL_VERSION` 1.4.1). LTV·DSCR·
금융비용부담은 여러 입력값이 얽힌 계산값이라 이번엔 손대지 않음 — 이미 6/6 사례가 정확한 상태라
게이트 자체를 다시 건드릴 필요는 없었고, 순수 항목 하나만 최소 수정. 전체 테스트(`test-scoring.mjs`
`test-pf-cases.mjs`) 통과, 회귀 없음(case-001/002 성공 사례도 여전히 일치, 등급만 소폭 하향
조정—AA-→A, A→BBB+, 더 정직해진 결과).
